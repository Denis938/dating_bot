# Архитектура Dating Bot

## 1. Обзор системы

### Тип архитектуры
**Модульный монолит** с возможностью выделения сервисов в будущем.

### Почему не микросервисы?
- ✅ Быстрый старт (нет overhead на orchestration)
- ✅ Проще деплой и мониторинг
- ✅ Нет проблем с distributed transactions
- ✅ Легче рефакторинг на ранней стадии

### Почему не чистый монолит?
- ✅ Четкие границы модулей
- ✅ Возможность выделения сервисов позже
- ✅ Разные команды могут работать независимо

## 2. Технологический стек

### Backend
```
├── Язык: Python 3.11+
├── Фреймворк: FastAPI
├── ORM: SQLAlchemy 2.0 (Async)
├── Миграции: Alembic
├── Валидация: Pydantic v2
└── Тесты: pytest + httpx
```

### База данных
```
├── Основная БД: PostgreSQL 15+
├── Кеш: Redis 7+
└── Object Storage: S3-compatible (MinIO / AWS S3)
```

### Client
```
├── Telegram Bot: aiogram 3.x
├── Web App: React + TypeScript (опционально)
└── WebSocket: Socket.io / native WS
```

### Infrastructure
```
├── Контейнеризация: Docker + Docker Compose
├── Message Queue: Redis Streams / RabbitMQ
└── Reverse Proxy: Nginx
```

```
┌─────────────────────────────────────────────────────────┐

### 6.2 Matching Algorithm
```
┌─────────────────────────────────────────────────────────┐

│     ├── +10: Online now                                 │

## 7. Безопасность

### 7.1 Защита данных
- HTTPS для всех соединений
- JWT токены с коротким временем жизни
- Пароли: bcrypt с salt
- SQL injection защита: SQLAlchemy ORM

### 7.2 Rate Limiting
```
┌────────────────────────────────────────┐
│         Rate Limits                    │
├────────────────────────────────────────┤
│  Auth endpoints:    5 / мин           │
│  Like endpoints:    30 / мин          │
│  Chat endpoints:    60 / мин          │
│  General API:       100 / мин         │
└────────────────────────────────────────┘
```

### 7.3 Валидация контента
- Фото: проверка типа, размера, NSFW фильтр
- Сообщения: фильтрация мата, спам защита
- Профиль: валидация возраста (18+)
